# TstPlatform 原型验证计划

> 目标：验证TSMaster COM API可行性，评估Vector脚本迁移成本
> 时间：2026-03-15 ~ 2026-04-30
> 方案：并行推进（盘点与原型同步进行）

---

## 一、项目概览

### 1.1 背景

- **存量资产**：几百个Vector脚本，涵盖诊断/刷写/网络管理/安全访问
- **工具现状**：现有TSMaster License，无需额外采购
- **业务诉求**：高频测试需稳定支持，迁移风险需提前评估

### 1.2 目标范围

| 功能域 | 验证内容 | 优先级 |
|--------|----------|--------|
| **UDS诊断** | $10会话控制、$22读写、$2E写入、$19 DTC | P0 |
| **网络管理** | NM状态检测、休眠唤醒 | P0 |
| **软件刷写** | $34请求下载、$36数据传输、$37退出传输 | P0 |
| **安全访问** | $27安全解锁（Seed-Key） | P0 |
| **信号测试** | 周期性报文、信号读写 | P1 |

### 1.3 成功标准

```
✅ Go标准（满足3项以上）：
   - UDS诊断服务响应时间与Vector差异<20%
   - 刷写流程可稳定完成（成功率>95%）
   - 安全解锁算法正确执行
   - 连续8小时运行无异常
   - 试点用例迁移工作量<原脚本开发的1.5倍

❌ No-Go标准（满足1项即停）：
   - 核心诊断服务无法正确执行
   - 刷写性能比Vector慢50%以上
   - 安全访问算法不支持
   - COM API存在无法绕过的稳定性问题
```

## 三、详细任务说明

### Task-1: Vector脚本盘点与分类

**输入**：几百个CAPL脚本文件

**输出**：
```
docs/prototype/
├── scripts_inventory.md      # 脚本清单（表格）
├── migration_matrix.md       # 迁移优先级矩阵
└── pilot_selection.md        # 试点用例选定及理由
```

**盘点维度**：

| 维度 | 说明 | 示例值 |
|------|------|--------|
| script_id | 脚本编号 | VEC-001 |
| name | 脚本名称 | 诊断会话控制测试 |
| category | 功能分类 | 诊断/刷写/NM/安全/信号 |
| complexity | 复杂度 | 高/中/低 |
| capl_functions | 使用的CAPL函数 | setSignal, diagRequest |
| vector_features | Vector特有功能 | Test Module, Panel |
| uds_services | 涉及的UDS服务 | $10, $22, $2E |
| estimated_effort | 预估迁移人天 | 2天/5天/10天 |
| risk_level | 技术风险 | 高/中/低 |
| migration_phase | 建议迁移阶段 | 试点/推广/暂缓 |

**试点用例选取标准**（选出5-8个）：
1. **代表性**：覆盖诊断/刷写/NM/安全四大领域
2. **可行性**：复杂度中等，避免过于复杂的场景
3. **价值**：高频使用的核心测试场景
4. **风险可控**：不涉及Vector特有高级功能

### Task-3: Vector到TSMaster适配层设计

**设计目标**：提供统一的抽象接口，降低上层代码的工具依赖性

**架构设计**：

```python
# 统一测试接口（抽象层）
class TestAdapter(ABC):
    @abstractmethod
    def connect(self, config): pass

    @abstractmethod
    def set_signal(self, name, value): pass

    @abstractmethod
    def get_signal(self, name): pass

    @abstractmethod
    def diag_request(self, service, payload): pass

    @abstractmethod
    def flash_download(self, address, data): pass

    @abstractmethod
    def security_unlock(self, level, algorithm): pass

# Vector实现
class VectorAdapter(TestAdapter):
    # 基于CANoe COM API实现
    pass

# TSMaster实现
class TSMasterAdapter(TestAdapter):
    # 基于TSMaster COM API实现
    pass
```

**CAPL函数映射表（示例）**：

| CAPL函数 | TSMaster等效 | 说明 |
|----------|-------------|------|
| `setSignal()` | `set_signal()` | 信号写入 |
| `getSignal()` | `get_signal()` | 信号读取 |
| `testWaitForTimeout()` | `time.sleep()` | 延时等待 |
| `diagRequest()` | `diag_request()` | 诊断请求 |
| `diagGetResponse()` | `get_diag_response()` | 诊断响应 |

## 五、风险与应对
